## 2. Headers:

At the minimum there must be a population and cohort header in each file. All others are optional and will over-write options selected in the interface.

Headers must be separated by commas and spaces outside of descriptions in quotes are not allowed.

A header example:

 > pop=OTH,cohort="Description of the cohort",license=CC0,resolution=G

### 2.1 Population

> pop=POPULATION_TAG

The population is a mandatory header, and to prevent duplicates, must use a population tag already found in the database. To add a population, please see the Options tab.

### 2.2 Cohort

> cohort="Description of the cohort"

The cohort is a mandatory header. You have 255 characters available to you to describe your cohort. 

### 2.3 License

> license=CC0

The license is an optional header. If it is not provided, the license selected on the main page will be applied. If provided, the license in the header will override the option selected on the main page. There are 7 license options:

- CC0: Creative Commons no rights reserved.
